Mapping Reductions in Computability Theory

Thu Apr 03 2025

Computability Theory

Introduction

In computability theory, it can be somewhat tedious to analyze the decidability and recognizability of each decision problem from scratch—especially when the problems are closely related. For example, we already know that the Halting Problem

HALT={⟨y,x⟩∈Σ∗×Σ∗∣y=code(M) and M halts on x}\text{HALT} = \{ \langle y, x \rangle \in \Sigma^* \times \Sigma^* \mid y = \text{code}(M) \text{ and } M \text{ halts on } x \}HALT={⟨y,x⟩∈Σ∗×Σ∗∣y=code(M) and M halts on x}

is undecidable. Now suppose the problem is slightly changed: does a given Turing machine halt on all inputs? Or does it halt on the empty input? Are these problems decidable?

In such cases, a general strategy for proving undecidability is:

  1. Assume there exists a Turing machine MMM that decides the target problem.
  2. Use MMM to construct a machine MHM_HMH​ that decides HALT\text{HALT}HALT.
  3. Contradiction—since HALT\text{HALT}HALT is known to be undecidable.

Abstracting this idea, to prove that a problem AAA is undecidable, we assume that AAA is decidable and show that this would imply the decidability of another problem BBB that is already known to be undecidable. Therefore, AAA cannot be decidable. Mapping reduction provides a unified and formal method for solving such problems.

Definition of Mapping Reduction

The idea of a mapping reduction is to transform one problem into another. It is formally defined as follows:

Let L1L_1L1​ and L2L_2L2​ be languages over alphabets Σ\SigmaΣ and Γ\GammaΓ, respectively. A mapping reduction from L1L_1L1​ to L2L_2L2​ is a computable total function f:Σ∗→Γ∗f: \Sigma^* \to \Gamma^*f:Σ∗→Γ∗, such that

x∈L1  ⟺  f(x)∈L2.x \in L_1 \iff f(x) \in L_2.x∈L1​⟺f(x)∈L2​.

If such a function fff exists, we write L1≤mL2L_1 \leq_m L_2L1​≤m​L2​.

A computable total function means that there exists a Turing machine which computes fff, and it halts on every input x∈Σ∗x \in \Sigma^*x∈Σ∗. In other words, the function always produces an output rather than looping indefinitely.

The condition x∈L1  ⟺  f(x)∈L2x \in L_1 \iff f(x) \in L_2x∈L1​⟺f(x)∈L2​ is equivalent to:

  • if x∈L1x \in L_1x∈L1​, then f(x)∈L2f(x) \in L_2f(x)∈L2​, and
  • if x∉L1x \notin L_1x∈/L1​, then f(x)∉L2f(x) \notin L_2f(x)∈/L2​.

In other words, YES-instances of the decision problem for L1L_1L1​ are mapped to YES-instances of the problem for L2L_2L2​, and NO-instances of L1L_1L1​ are mapped to NO-instances of L2L_2L2​.

In fact, a mapping reduction is a concrete realization of the step 2 in the general proof strategy for undecidability we mentioned earlier—it directly transforms one problem into another. The hypothetical machine MMM is only used at the final step to reach a contradiction.

It can be noted that mapping reduction is, in fact, an independent process—not directly tied to decidability or recognizability—but it can be used to prove both. This proof technique also plays an important role in many other areas, such as complexity theory.

Example

Now, when we attempt to prove that a language LLL is undecidable, it suffices to show that there exists a function fff such that

HALT≤mL,\text{HALT} \leq_m L,HALT≤m​L,

i.e., fff reduces HALT\text{HALT}HALT to LLL. If LLL were decidable by some Turing machine MMM, then HALT\text{HALT}HALT could also be decided by composing MMM with fff (as shown below), leading to a contradiction. Therefore, LLL must be undecidable.

Mapping Reduction in Computability Theory

If L1≤mL2L_1 \leq_m L_2L1​≤m​L2​, then there exists a Turing machine that transforms the decision problem of L1L_1L1​ into the decision problem of L2L_2L2​. This means that, in terms of computability, if we can solve L2L_2L2​, then we can solve L1L_1L1​. In other words, if we know the hardness of L1L_1L1​, then L2L_2L2​ is at least as hard as L1L_1L1​. Conversely, if we know the hardness of L2L_2L2​, then L1L_1L1​ is at most as hard as L2L_2L2​.

Formally, for any languages L1L_1L1​ and L2L_2L2​, mapping reduction has the following properties:

  • If L2L_2L2​ is decidable and L1≤mL2L_1 \leq_m L_2L1​≤m​L2​, then L1L_1L1​ is decidable. (The following Turing machine decides L1L_1L1​.)

  • If L1L_1L1​ is undecidable and L1≤mL2L_1 \leq_m L_2L1​≤m​L2​, then L2L_2L2​ is undecidable. (Logically equivalent to the above)

  • If L2L_2L2​ is decidable and L1L_1L1​ is undecidable, then L1̸≤mL2L_1 \not\leq_m L_2L1​≤m​L2​. (Also logically equivalent)

For recognizability:

  • If L2L_2L2​ is recognizable and L1≤mL2L_1 \leq_m L_2L1​≤m​L2​, then L1L_1L1​ is recognizable. (The following Turing machine recognize L1L_1L1​.)
  • If L1L_1L1​ is not recognizable and L1≤mL2L_1 \leq_m L_2L1​≤m​L2​, then L2L_2L2​ is not recognizable.
  • If L2L_2L2​ is recognizable and L1L_1L1​ is not recognizable, then L1̸≤mL2L_1 \not\leq_m L_2L1​≤m​L2​.

The Empty String Halting Problem

The Empty String Halting (ETH) problem asks whether a given Turing machine halts on the empty string. This problem may sound much simpler than the general Halting Problem, since the input is restricted to just one specific case. It is formally defined as:

ETH={x∈Σ∗∣x=code(M) and M halts on ε}.\text{ETH} = \{ x \in \Sigma^* \mid x = \text{code}(M) \text{ and } M \text{ halts on } \varepsilon \}.ETH={x∈Σ∗∣x=code(M) and M halts on ε}.

However, we will prove that HALT≤mETH\text{HALT} \leq_m \text{ETH}HALT≤m​ETH, thereby showing that ETH\text{ETH}ETH is undecidable.

To prove that HALT≤mETH\text{HALT} \leq_m \text{ETH}HALT≤m​ETH, we need to construct a computable function fff such that:

⟨y,x⟩∈HALT  ⟹  f(⟨y,x⟩)∈ETH⟨y,x⟩∉HALT  ⟹  f(⟨y,x⟩)∉ETH\begin{aligned} \langle y, x \rangle \in \text{HALT} &\implies f(\langle y, x \rangle) \in \text{ETH} \\ \langle y, x \rangle \notin \text{HALT} &\implies f(\langle y, x \rangle) \notin \text{ETH} \end{aligned}⟨y,x⟩∈HALT⟨y,x⟩∈/HALT​⟹f(⟨y,x⟩)∈ETH⟹f(⟨y,x⟩)∈/ETH​

  • Check whether ⟨y,x⟩\langle y, x \rangle⟨y,x⟩ is a valid encoding—that is, whether yyy is the encoding of some Turing machine.
    If not, let f(⟨y,x⟩)=yf(\langle y, x \rangle) = yf(⟨y,x⟩)=y. Clearly, y∉ETHy \notin \text{ETH}y∈/ETH.

  • Otherwise, let M=decode(y)M = \text{decode}(y)M=decode(y), and define f(⟨y,x⟩)=code(MM,x)f(\langle y, x \rangle) = \text{code}(M_{M,x})f(⟨y,x⟩)=code(MM,x​), where MM,xM_{M,x}MM,x​ is constructed as follows:

    • On any non-empty input, MM,xM_{M,x}MM,x​ enters an infinite loop.
    • On the empty input, MM,xM_{M,x}MM,x​ simulates MMM on input xxx.

Now, for a valid encoding ⟨y,x⟩\langle y, x \rangle⟨y,x⟩:

  • If ⟨y,x⟩∈HALT\langle y, x \rangle \in \text{HALT}⟨y,x⟩∈HALT, then MM,xM_{M,x}MM,x​ halts on the empty input, so f(⟨y,x⟩)=code(MM,x)∈ETHf(\langle y, x \rangle) = \text{code}(M_{M,x}) \in \text{ETH}f(⟨y,x⟩)=code(MM,x​)∈ETH.

The rule “on any non-empty input, MM,xM_{M,x}MM,x​ enters an infinite loop” is only used to define the behavior on non-empty inputs and ensure the computability of fff.

Limits of Mapping Reduction from Easy Problems

In the example above, we reduced a provenly hard decision problem (HALT\text{HALT}HALT) to a seemingly simpler one (ETH\text{ETH}ETH) via a mapping reduction, in order to demonstrate that the latter is equally hard. We also know that we can reduce a seemingly hard problem to a known easy one in order to solve it directly. But what happens if we reduce a provenly easy problem to another problem?

The answer is that we only learn that the latter is at least as “hard” as the former—which is not very helpful. In fact, for sufficiently simple decision problems, such as decidable languages, it is possible to reduce them to any nontrivial language LLL, that is, any L≠Σ∗L \neq \Sigma^*L=Σ∗ and L≠∅L \neq \emptysetL=∅.

For any decidable language L1L_1L1​ and any nontrivial language L2L_2L2​, suppose MMM decides L1L_1L1​. Since L2L_2L2​ is nontrivial, we can find strings y∈L2y \in L_2y∈L2​ and z∉L2z \notin L_2z∈/L2​. Now consider the function fff defined as follows:

f(x)={yif M accepts xzif M rejects xf(x) = \begin{cases} y & \text{if } M \text{ accepts } x \\ z & \text{if } M \text{ rejects } x \end{cases}f(x)={yz​if M accepts xif M rejects x​

If x∈L1x \in L_1x∈L1​, then MMM accepts xxx, so f(x)=y∈L2f(x) = y \in L_2f(x)=y∈L2​.
If x∉L1x \notin L_1x∈/L1​, then MMM rejects xxx, so f(x)=z∉L2f(x) = z \notin L_2f(x)=z∈/L2​.

This kind of construction relies on the assumption that L1L_1L1​ is simple enough—that is, there exists a Turing machine MMM that decides it.

Rice’s Theorem

Finally, we use mapping reduction to prove Rice’s Theorem.

Any nontrivial property about the language L(M)L(M)L(M) recognized by a Turing machine MMM is undecidable.
— Rice’s Theorem

Nontrivial means that not all recognizable languages L(M)L(M)L(M) have the property, and not none of them do either.

First, we need to understand what a property about a language L(M)L(M)L(M) means. Formally, for any nontrivial set of Turing-recognizable languages S⊂{L(M)∣M is a Turing machine}\mathcal{S} \subset \{ L(M) \mid M \text{ is a Turing machine} \}S⊂{L(M)∣M is a Turing machine}, with S≠∅\mathcal{S} \neq \emptysetS=∅, we define the language property:

P={code(M)∣L(M)∈S}P = \{ \text{code}(M) \mid L(M) \in \mathcal{S} \}P={code(M)∣L(M)∈S}

Note that PPP is a set of Turing machine codes, not a set of languages. This refers to a property of the Turing machine MMM, but one that depends only on the language it recognizes. A language L(M)L(M)L(M) is a subset of Σ∗\Sigma^*Σ∗, and any description of this set constitutes a language property. For example: whether L(M)L(M)L(M) is finite, or whether the string “abc” belongs to L(M)L(M)L(M), are both properties of the language L(M)L(M)L(M).

In other words, even if two Turing machines M1M_1M1​ and M2M_2M2​ are different, as long as they recognize the same language—that is, L(M1)=L(M2)L(M_1) = L(M_2)L(M1​)=L(M2​)—then we have P(M1)=P(M2)P(M_1) = P(M_2)P(M1​)=P(M2​).

In contrast, properties that are related to the implementation of the Turing machine MMM are not considered language properties. For instance: whether MMM halts within 100 steps, or whether the size of its state set is less than 10.

As a result, Rice’s Theorem cannot be used to prove the undecidability of the Halting Problem, since that is a property of the machine, not of the language it recognizes.

Proof

Fix a property PPP. Assume that the empty language does not have this property, i.e., P(code(M∅))=0P(\text{code}(M_\emptyset)) = 0P(code(M∅​))=0. Since PPP is nontrivial, there exists some language L(MP)L(M_P)L(MP​) such that P(code(MP))=1P(\text{code}(M_P)) = 1P(code(MP​))=1. Now we construct a function fff as follows:

  • If ⟨y,x⟩\langle y, x \rangle⟨y,x⟩ is not a valid encoding (i.e., yyy is not the encoding of any Turing machine), then let f(⟨y,x⟩)=code(M∅)f(\langle y, x \rangle) = \text{code}(M_\emptyset)f(⟨y,x⟩)=code(M∅​).
  • Otherwise, let M=decode(y)M = \text{decode}(y)M=decode(y), and define f(⟨y,x⟩)=code(MM,x)f(\langle y, x \rangle) = \text{code}(M_{M,x})f(⟨y,x⟩)=code(MM,x​), where MM,xM_{M,x}MM,x​ is constructed as follows:
    1. On any input zzz, MM,xM_{M,x}MM,x​ first simulates MMM on input xxx;
    2. If MMM halts, then it simulates MPM_PMP​ on input zzz;
    3. If MPM_PMP​ accept the input zzz, then MM,xM_{M,x}MM,x​ accepts the input zzz.

Now, for a valid encoding ⟨y,x⟩\langle y, x \rangle⟨y,x⟩:

  • If ⟨y,x⟩∈HALT\langle y, x \rangle \in \text{HALT}⟨y,x⟩∈HALT, then MMM halts on input xxx. Then, if MPM_PMP​ accepts some input zzz, so does MM,xM_{M,x}MM,x​. In our construction, one of the key step is that when ⟨y,x⟩∉HALT\langle y, x \rangle \notin \text{HALT}⟨y,x⟩∈/HALT, the machine MM,xM_{M,x}MM,x​ loops on every input, and

However, the assumption P(code(M∅))=0P(\text{code}(M_\emptyset)) = 0P(code(M∅​))=0 is something we made. What if instead P(code(M∅))=1P(\text{code}(M_\emptyset)) = 1P(code(M∅​))=1?

In that case, we can simply consider the complement property ¬P\neg P¬P. Then we have shown that

{code(M)∣¬P(code(M))=1}\{ \text{code}(M) \mid \neg P(\text{code}(M)) = 1 \}{code(M)∣¬P(code(M))=1}

is undecidable, which is logically equivalent to saying that

{code(M)∣P(code(M))=1}\{ \text{code}(M) \mid P(\text{code}(M)) = 1 \}{code(M)∣P(code(M))=1}

is undecidable.

Conclusion

At this point, we have developed an understanding of mapping reduction as a tool for transforming one decision problem into another. Mapping reductions are not only used in computability theory, but also play an important role in complexity theory, where they are applied in a similar way.

We then used mapping reductions to prove the undecidability of the Empty String Halting (ETH\text{ETH}ETH) problem and Rice’s Theorem, demonstrating that mapping reduction is a general and powerful technique.
